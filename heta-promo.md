大家好，今天介绍一下我们团队的新开源项目：**Heta** —— 专为 AI Agent 设计的知识管理平台。

---

**🤔 Agent 的瓶颈在哪里？**

我们给 Agent 配上了文档库、记忆模块、知识图谱……工具越堆越多，但问题也随之而来：

这些知识从哪来、存在哪、怎么用——没有人统一管。

PDF 丢向量库，对话记忆单独存，图谱又是另一套系统，查询时 Agent 不知道该问谁、该信谁。更麻烦的是，用户还得自己判断"这份知识该进哪个库"——向量库？图数据库？关系型数据库？这本不该是用户操心的问题。

**缺的不是更多的知识，而是一个能真正接管这一切、让知识各归其位、按需调度的统一平台。我们相信，对知识的正确管理是 Agent 实现自我进化的必要前提——只有知识被有效组织、持续更新，Agent 才能真正学到新内容，在通用能力之上走向专业化，在科学探索等前沿领域不断沉淀、形成突破。**

---

**💎 Heta 是什么？**

Heta 是一套专为 Agent 设计的知识管理平台，整合多种类型数据库，为 Agent 提供从外部知识获取到知识积累再到基于知识进行推理与生成的的全栈能力。

它包含三个层次：

**📚 HetaDB — 外部知识的获取与统一管理**
支持 PDF、HTML、图片、表格、Markdown 等多格式文件解析入库，知识自动分配至合适的存储结构，无需用户手动选择。提供从简单向量检索到多跳推理的五种查询策略，并支持源文档溯源，帮助 Agent 实现更可靠的 Agentic Search。

**🧠 HetaMem — Agent 自身知识的积累与演化**
双模式记忆设计：MemoryVG 处理高频、碎片化的情景记忆（对话事实、用户偏好、上下文），MemoryKB 则构建随 Agent 持续演化的长期知识图谱。知识随使用不断沉淀，Agent 越用越懂你。

**⚙️ HetaGen — 基于知识的结构化生成**（早期阶段）
在已有知识库基础上进行归纳与扩展，生成结构化表格、知识层次体系，并支持 Text-to-SQL 查询。
