Attention Mechanism in Machine Learning

Source: arXiv:1409.0473 (Bahdanau et al., 2014) and arXiv:1706.03762 (Vaswani et al., 2017)

1. Overview

The attention mechanism is a technique in machine learning that allows models to selectively focus on relevant parts of the input when producing each part of the output, rather than compressing all input information into a single fixed-length vector.

Attention was first introduced for neural machine translation by Dzmitry Bahdanau, Kyunghyun Cho, and Yoshua Bengio in their 2014 paper "Neural Machine Translation by Jointly Learning to Align and Translate" (arXiv:1409.0473), accepted at ICLR 2015 as an oral presentation.

Key insight (Bahdanau et al.): "The use of a fixed-length vector is a bottleneck in improving the performance of this basic encoder-decoder architecture." The attention mechanism allows models to automatically (soft-)search for parts of a source sentence that are relevant to predicting a target word.

2. Types of Attention

Type Description Key Paper Year
Soft / Global Attention Attends to all input positions with a weighted combination Bahdanau et al. 2014
Hard Attention Selects a single input position; non-differentiable, uses REINFORCE Xu et al. (Show, Attend and Tell) 2015
Self-Attention Each position attends to all positions in the same sequence Vaswani et al. (Transformer) 2017
Multi-Head Attention h parallel attention heads applied simultaneously Vaswani et al. (Transformer) 2017
Cross-Attention Queries from one sequence attend to keys/values from another Vaswani et al. (Transformer) 2017

3. Scaled Dot-Product Attention

The Transformer (Vaswani et al., 2017) introduced scaled dot-product attention, defined as:

Attention(Q, K, V) = softmax( Q·KT / √dk ) · V

where Q is the matrix of queries, K is the matrix of keys, V is the matrix of values, and dk is the dimension of the key vectors. The scaling factor 1/√dk is applied to prevent large dot products from pushing the softmax function into regions with extremely small gradients.

Within the Transformer team, Noam Shazeer proposed scaled dot-product attention, multi-head attention, and the parameter-free position representation.

4. Multi-Head Attention

Multi-head attention allows the model to jointly attend to information from different representation subspaces at different positions. The Transformer uses h = 8 parallel attention heads:

MultiHead(Q, K, V) = Concat(head1, ..., headh) · WO
where headi = Attention(Q·WiQ, K·WiK, V·WiV)

With dmodel = 512 and h = 8, each head operates in a subspace of dimension dk = dv = 64.

5. Self-Attention vs. Recurrence: Computational Complexity

Layer Type Complexity per Layer Sequential Operations Maximum Path Length
Self-Attention O(n² · d) O(1) O(1)
Recurrent (RNN) O(n · d²) O(n) O(n)
Convolutional O(k · n · d²) O(1) O(logk(n))

Self-attention connects all positions with a constant number of sequential operations, whereas RNNs require O(n) sequential operations — a key advantage for parallelization during training.

6. Applications

  • Machine Translation: Original application of Bahdanau attention (EN-FR).
  • Image Captioning: "Show, Attend and Tell" (Xu et al., 2015) applied attention to visual features for caption generation.
  • Question Answering: Attention over passages to locate answer spans.
  • Pre-trained Language Models: BERT, GPT-2, GPT-3 and all modern LLMs are based on the Transformer architecture with multi-head self-attention.
  • Computer Vision: Vision Transformer (ViT) applies self-attention directly to sequences of image patches.
